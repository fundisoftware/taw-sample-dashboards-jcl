Introduction

IBM Transaction Analysis Workbench for z/OS (colloquially known as Workbench or TAW) is an operations analytics tool that can forward many types of z/OS-based logs to analytics platforms such as Splunk and Elastic.

You can get sample Splunk and Elastic dashboards that present log data forwarded by Workbench.

This documentation presents JCL for using Workbench to forward your own log data to those sample dashboards.

In particular, this documentation describes:

  • Log record types required by each dashboard
  • Fields that the dashboards require from those record types

For an introduction to the sample dashboards, watch this video:

The video states that the fields in the JCL generated by the Workbench ISPF dialog match the fields required by the sample dashboards. However, as of February 2018, that statement is not true. The fields in the JCL generated by the dialog are intended only as a starting point for operations analytics; they do not match the specific fields required by the sample dashboards. To forward your own logs to the sample dashboards, specify the fields in the JCL presented in this documentation.
